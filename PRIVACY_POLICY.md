# Privacy Policy for NYANCAT Discord Bot

**Last Updated:** November 2024

## Introduction

NYANCAT ("we", "our", or "the bot") is a Discord bot designed to provide community management and engagement features. This Privacy Policy explains how we collect, use, and protect your data when you use our bot.

## Data We Collect

### 1. User Data
- **User ID**: Stored to identify users for various features (rank system, economy, etc.)
- **Server ID**: Stored to associate data with specific Discord servers
- **Username and Nickname**: Used for display purposes in commands and logs

### 2. Message Content Data
- **Message ID**: Unique identifier for messages
- **Channel ID**: Identifier of the channel where messages were sent
- **User ID**: Identifier of the user who sent the message
- **Message Content**: The actual text content of messages
- **Attachment Information**: File names, URLs, and metadata of attachments
- **Timestamp**: When the message was created and modified

### 3. Activity Data
- **XP and Level Data**: Chat XP, voice XP, levels for rank system
- **Economy Data**: Virtual currency balances, transaction history, stock holdings
- **Attendance Data**: Daily check-in records and statistics
- **Rank Data**: User rankings and statistics

### 4. Server Management Data
- **Warning Records**: Warning counts and history for moderation
- **Ticket Records**: Support ticket conversations and records
- **Log Data**: Server activity logs (chat, voice, user, channel, role, emoji, invite, moderation)

### 5. Presence Data
- **Online Status**: Real-time user status (online, idle, do not disturb, offline)
- **Activity Information**: Games being played, custom statuses

**Note:** Presence data is NOT stored. It is only retrieved in real-time when commands are executed and displayed immediately.

## How We Use Your Data

### 1. Core Bot Functionality
- **Rank System**: Calculate XP, levels, and rankings based on message activity
- **Economy System**: Manage virtual currency, transactions, and virtual stock trading
- **Attendance System**: Track daily check-ins and maintain attendance records
- **Moderation**: Issue warnings, track moderation actions, and maintain server safety

### 2. Server Administration
- **Logging**: Record server activity for administrators to monitor and review
- **Ticket System**: Maintain support ticket conversations and records
- **Welcome/Goodbye Messages**: Send personalized messages when users join or leave servers

### 3. User Experience
- **Information Commands**: Display user and server information, including status and activity
- **CAPTCHA Verification**: Verify users by comparing input with verification codes
- **Custom Features**: Provide personalized experiences based on user data

## Data Storage

### Storage Location
- Data is stored in a secure database (Supabase) hosted off-platform
- Message content and logs are stored for server administration purposes
- User activity data (XP, economy, etc.) is stored to maintain bot functionality

### Data Retention
- **Message Logs**: Stored indefinitely for server administration and moderation purposes
- **User Data**: Retained as long as the user is in servers where the bot is active
- **Economy Data**: Retained to maintain user progress and virtual assets
- **Ticket Records**: Retained for support and record-keeping purposes

### Data Deletion
- Users can request data deletion by contacting the bot developer
- Server administrators can request deletion of server-specific data
- Data will be deleted within 30 days of a valid deletion request
- Some data may be retained if required by law or for legitimate business purposes

## Data Access and Sharing

### Who Can Access Your Data
- **Server Administrators**: Can access message logs and server-specific data through the bot's logging system
- **Bot Developers**: Have access to the database for maintenance and support purposes
- **Users**: Can view their own data through bot commands (rank, balance, etc.)

### Data Sharing
- We do not sell, trade, or rent your personal data to third parties
- We do not share your data with other Discord bots or services without your explicit consent
- Data may be shared if required by law or to protect our rights and safety

## User Rights and Controls

### Opt-Out Options
- **Message Content**: Users can opt out by disabling message content access in Discord privacy settings. However, some bot features (rank system, CAPTCHA verification) may not function properly.
- **Presence Data**: Users can opt out by disabling status sharing in Discord privacy settings. The bot respects these settings and will not display status information if users have opted out.

### Data Access
- Users can view their own data through bot commands:
  - `/랭크` - View rank and XP data
  - `/잔액` - View economy balance
  - `/출석 통계` - View attendance statistics
  - `/경고 확인` - View warning records

### Data Deletion Requests
- Users can request deletion of their data by contacting the bot developer
- Include your Discord User ID and the servers where you want data deleted
- We will process deletion requests within 30 days

## Security

We take data security seriously and implement appropriate measures to protect your data:
- Secure database connections (encrypted)
- Access controls and authentication
- Regular security updates and monitoring
- Limited access to data on a need-to-know basis

However, no method of transmission over the internet or electronic storage is 100% secure. While we strive to protect your data, we cannot guarantee absolute security.
